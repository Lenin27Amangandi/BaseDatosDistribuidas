# Contexto completo del proyecto: Integración SQL Server MEC + SIS mediante Linked Server

## Descripción general

Se tienen dos bases de datos independientes en SQL Server:

- Base de datos **MEC** (Facultad de Mecánica).
- Base de datos **SIS** (otro sistema/facultad).

Se configuró un **Linked Server** para que MEC pueda consultar información de SIS mediante:

```
[WIN-RJBV795UJE8].[SIS].[dbo].[tabla]
```

El objetivo es consultar estudiantes que tengan materias matriculadas en ambas facultades.

La relación principal entre los sistemas se realiza mediante:

```
idmat
```

Es decir, una materia se considera común si existe el mismo `idmat` en:

- `matriculaMEC`
- `matriculaSIS`

---

# Problema inicial encontrado

Se intentó crear una vista:

```sql
USE SIS

CREATE VIEW V_matricula
AS
SELECT *
FROM [WIN-JEAMMFTDC9V].[MEC].[dbo].[matriculaMEC]

UNION ALL

SELECT *
FROM [WIN-RJBV795UJE8].[SIS].[dbo].[matriculaSIS]
```

Pero SQL Server mostró:

```
Msg 111, Level 15, State 1
'CREATE VIEW' must be the first statement in a query batch.
```

## Causa

En SQL Server, `CREATE VIEW` debe ser la primera instrucción del lote.

`USE SIS` estaba antes del `CREATE VIEW`.

## Solución

Separar los bloques con `GO`:

```sql
USE SIS;
GO

CREATE VIEW dbo.V_matricula
AS
SELECT *
FROM [WIN-JEAMMFTDC9V].[MEC].[dbo].[matriculaMEC]

UNION ALL

SELECT *
FROM [WIN-RJBV795UJE8].[SIS].[dbo].[matriculaSIS];
GO
```

# Consulta inicial para encontrar materias comunes

La consulta utilizada fue:

```sql
USE MEC;
GO

WITH MateriasComunes AS
(
    SELECT DISTINCT mMEC.idmat
    FROM dbo.matriculaMEC AS mMEC
    WHERE EXISTS
    (
        SELECT 1
        FROM [WIN-RJBV795UJE8].[SIS].[dbo].[matriculaSIS] AS mSIS
        WHERE mSIS.idmat = mMEC.idmat
    )
)

SELECT
    eMEC.nombre,
    eMEC.apellido,
    mMEC.idmat,
    'MEC' AS origen
FROM dbo.matriculaMEC AS mMEC
INNER JOIN dbo.estudianteMEC AS eMEC
    ON eMEC.CC = mMEC.CC
INNER JOIN MateriasComunes AS mc
    ON mc.idmat = mMEC.idmat

UNION ALL

SELECT
    eSIS.nombre,
    eSIS.apellido,
    mSIS.idmat,
    'SIS' AS origen
FROM [WIN-RJBV795UJE8].[SIS].[dbo].[matriculaSIS] AS mSIS
INNER JOIN [WIN-RJBV795UJE8].[SIS].[dbo].[estudianteSIS] AS eSIS
    ON eSIS.CC = mSIS.CC
INNER JOIN MateriasComunes AS mc
    ON mc.idmat = mSIS.idmat;
GO
```

---

# Problema: la consulta devolvía vacío

## Causa

No existía ninguna materia (`idmat`) registrada en ambos sistemas.

La consulta dependía de esta condición:

```sql
mMEC.idmat = mSIS.idmat
```

Por lo tanto, debía existir una materia igual en ambas bases.

Ejemplo:

MEC:

```
idmat = 003
```

SIS:

```
idmat = 003
```

```
REFERENCES materiaMEC(idmat,idfac)

```

# Conclusión

El problema inicial no era la consulta, sino que no existían datos relacionados entre ambos nodos.

La solución fue:

1. Crear estudiantes en ambos sistemas.
2. Usar la misma materia (`idmat=003`) en ambas facultades.
3. Crear matrículas asociadas.
4. Consultar mediante Linked Server usando `idmat` como campo común.

Con esto la integración MEC-SIS funciona correctamente.
